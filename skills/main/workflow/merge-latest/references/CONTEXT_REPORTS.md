# Reports and Context

`merge-latest` produces structured merge reports and state files. The report frontmatter envelope follows the `context-reports` shared schema (`skill`, `key`, `generated_at`, plus repo-convention `branch`/`commit` and merge-latest's `result`).

## Layout deviation (recorded decision)

The `context-reports` canonical layout is `{context_dir}/{report-type}/{key}.md`, organized by report type. `merge-latest` organizes by producing skill (`.agents/context/merge-latest/...`) so the report, state, and backups for a merge stay co-located with its resumable working state. This is an explicit, recorded deviation, not conformance. The repo-level layout question is deferred: `verify-branch` deviates the same way, and both should be conformed (or the convention revised) in one repo-level decision.

## Produced reports

### Merge report

```text
.agents/context/merge-latest/{target}-merge-report.md
```

Full record of the merge attempt: result, commits, files, conflicts, resolutions, validation output, confidence assessment, and next steps. Generated by `scripts/report.js`; see [REFERENCE.md](REFERENCE.md#report-format) for the shape.

### State file

```text
.agents/context/merge-latest/{target}/state.md
```

Working memory following the `checkpoint` skill's schema: branch inference, phase checklist, conflict status, resolutions, validation result, decisions. See [CHECKPOINTING.md](CHECKPOINTING.md).

### Backup

```text
.agents/context/merge-latest/backups/{target}-{timestamp}/
```

Snapshot of current HEAD before the merge attempt.

## Key placeholder

`{target}` is the target branch name (sanitized if needed).

## Consumed context

| Source | Location | Use |
|--------|----------|-----|
| Git metadata | local `.git` | branch history, commits, blame |
| `jira-adapter` skill | via dependency | Jira ticket scope for branch ticket keys |
| `token-resolver` skill | via dependency | adapter credential resolution |
| GitHub MCP | via harness | PR and commit context |

Missing sources degrade to git metadata with disclosure; see [CAPABILITIES.md](CAPABILITIES.md#degraded-enrichment-disclosure).
