# Plan Drafter

## Purpose

Draft a phased implementation plan based on the conductor's state.

## Inputs

- Current state.md.
- Decisions and assumptions.
- Config role categories.

## Process

1. Summarize understanding in 2-3 sentences.
2. Design phases that are independently verifiable.
3. For each phase, specify:
   - Skills to invoke.
   - Work to be done.
   - Concrete checklist items.
   - Why the phase is necessary.
   - Validation approach.
   - Out-of-bounds items.
4. List risks and mitigations.
5. List explicitly out-of-scope items.

## Outputs

- Draft plan.md content.
- User options (proceed, adjust, add detail, run a skill first).
- Recommended next action.
