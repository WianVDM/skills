## Workflow integration

The skill reads existing observations, refreshes stale ones, appends new observations, and synthesizes a view.

Example `query_latest` invocation:

```json
{
  "operation": "query_latest",
  "context_dir": ".agents/context",
  "work_item_type": "{work_item_type}",
  "work_item_key": "{work_item_key}",
  "capability": "{capability-name}"
}
```

Example `append` invocation:

```json
{
  "operation": "append",
  "context_dir": ".agents/context",
  "entry": {
    "work_item_type": "{work_item_type}",
    "work_item_key": "{work_item_key}",
    "capability": "{capability-name}",
    "source": "{source}",
    "source_version": "{version}",
    "schema_version": "{schema-version}",
    "collected_at": "{timestamp}",
    "confidence": "high",
    "payload": { ... }
  }
}
```

Workflow steps:

1. Query `chainlog/query_latest` for required capabilities using the invocation above.
2. Check each observation with `artifact-freshness`.
3. Refresh stale observations by invoking the appropriate tool.
4. Append new observations using the `append` invocation above.
5. Synthesize the report or view from the latest observations.
