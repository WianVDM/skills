# Skill Architecture Library — Philosophy

## Mission

To define the smallest set of fundamentals that make any skill great, and to document a modular architecture pattern that lets skills be composed from reusable building blocks, conductors, and wrappers. This workspace is both the specification and a living library of skills built on those principles.

## Core beliefs

1. **A skill is the smallest load-bearing shape.** It should do one thing well and carry only what it needs.
2. **Minimalism beats completeness.** A lean skill that is reliably invoked is better than a bloated skill that is ignored.
3. **Fundamentals are universal; architecture is opt-in.** Every skill needs identity, focus, and clarity. The library pattern is for skills that want to be composed, shared, and reused.
4. **The model is a tool user too.** Skills can be invoked by humans, but the model should be able to use them as first-class tools.
5. **Composition over monoliths.** Complex behavior should emerge from simple skills composed together, not from one skill that does everything.
6. **Refinement is continuous.** Skills and standards are polished over time as we learn what works.

## What this workspace is

- A place to document the core of what makes a skill great.
- A place to document a specific architecture pattern for composable, reusable skills.
- A place to develop skills that follow those principles.

## What this workspace is not

- A generic prompt library.
- A one-size-fits-all framework.
- A fixed, unchanging specification.
- A place for skills that do not know what they are for.

## How this workspace evolves

We start with the smallest possible fundamentals and the smallest possible architecture. We add patterns only when a recurring need proves they are necessary. We change existing skills when the standards change. We never keep a rule just because it was written down.
