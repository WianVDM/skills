# Skill standards

This is the wiki for the skills in this repo. It contains the rules, patterns, and reference material that turn the manifestos into working skills.

Start with the manifestos for the high-level ideas:

- [`docs/manifestos/philosophy.md`](../manifestos/philosophy.md)
- [`docs/manifestos/architecture.md`](../manifestos/architecture.md)

Then read the standards in this wiki:

| Section | Purpose |
|---|---|
| [`start/`](./start) | What a skill is, when to create one, examples. |
| [`fundamentals/`](./fundamentals) | Universal rules every skill must satisfy. |
| [`guides/`](./guides) | Task-oriented guides: write, review, audit, migrate. |
| [`patterns/`](./patterns) | Opt-in architecture patterns and portability. |
| [`reference/`](./reference) | Format, package, evaluation, glossary, and lookup tables. |

Pick a path:

- **Writing a skill:** `start/what-is-a-skill.md` → `fundamentals/types.md` → `guides/write-a-skill.md`
- **Reviewing a skill:** `reference/review-principles.md` → `guides/review-a-skill.md`
- **Auditing a skill:** `reference/audit-rubric.md` → `guides/audit-a-skill.md`
- **Migrating a skill:** `guides/migrate-a-skill.md`
