# Write a skill

1. Read [`fundamentals/core/what-is-a-skill/`](../fundamentals/core/what-is-a-skill/) and [`fundamentals/core/types/`](../fundamentals/core/types/).
2. Choose the right type: building block, conductor, or wrapper.
3. Draft the smallest viable `SKILL.md` using [`reference/format.md`](../reference/format.md) and [`fundamentals/core/structure/`](../fundamentals/core/structure/).
4. Declare dependencies and design the capability-to-tool strategy using [`fundamentals/architecture/dependencies-and-bundling.md`](../fundamentals/architecture/dependencies-and-bundling.md) and [`fundamentals/architecture/tooling-awareness.md`](../fundamentals/architecture/tooling-awareness.md).
5. Check [`fundamentals/core/common-mistakes/`](../fundamentals/core/common-mistakes/) before expanding.
6. If the skill is global, configurable, stateful, report-producing, reusable, or versioned, read the relevant pattern docs in [`patterns/`](../patterns).
7. Review against [`fundamentals/architecture/evaluation.md`](../fundamentals/architecture/evaluation.md). For practical testing, see [`reference/trigger-evals.md`](../reference/trigger-evals.md) and [`reference/context-budget.md`](../reference/context-budget.md).
