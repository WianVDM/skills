# Write a skill

1. Read [`start/what-is-a-skill.md`](../start/what-is-a-skill.md) and [`fundamentals/types.md`](../fundamentals/types.md).
2. Choose the right type: building block, conductor, or wrapper.
3. Draft the smallest viable `SKILL.md` using [`reference/format.md`](../reference/format.md) and [`fundamentals/structure.md`](../fundamentals/structure.md).
4. Declare dependencies and design the capability-to-tool strategy using [`fundamentals/dependencies-and-bundling.md`](../fundamentals/dependencies-and-bundling.md) and [`fundamentals/tooling-awareness.md`](../fundamentals/tooling-awareness.md).
5. Check [`fundamentals/common-mistakes.md`](../fundamentals/common-mistakes.md) before expanding.
6. If the skill is global, configurable, stateful, report-producing, reusable, or versioned, read the relevant pattern docs in [`patterns/`](../patterns).
7. Review against [`fundamentals/evaluation.md`](../fundamentals/evaluation.md). For practical testing, see [`reference/trigger-evals.md`](../reference/trigger-evals.md) and [`reference/context-budget.md`](../reference/context-budget.md).
