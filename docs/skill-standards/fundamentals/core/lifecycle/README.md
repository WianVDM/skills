# Skill lifecycle

**Layer:** universal fundamentals. **Mode:** reference.

A skill is a living document. It moves through distinct phases from initial idea to retirement. Treat each phase as intentional, not accidental.

---

## The phases

1. [`decide.md`](./decide.md) — confirm a skill is the right solution.
2. [`design.md`](./design.md) — choose type, form, invocation, and scope.
3. [`draft.md`](./draft.md) — write the minimal `SKILL.md` and supporting files.
4. [`validate.md`](./validate.md) — check structure and style before testing behavior.
5. [`test.md`](./test.md) — run trigger and behavioral evals.
6. [`iterate.md`](./iterate.md) — improve based on results and usage.
7. [`publish.md`](./publish.md) — install or publish the skill.
8. [`maintain.md`](./maintain.md) — review and remove sediment.
9. [`deprecate-or-split.md`](./deprecate-or-split.md) — retire or divide the skill.

---

## Lifecycle checklist

- [ ] Decided a skill is the right solution.
- [ ] Chosen type, form, invocation, and scope.
- [ ] Drafted a minimal `SKILL.md` and supporting files.
- [ ] Validated structure and style.
- [ ] Tested triggers and behavior against baselines.
- [ ] Iterated based on failures.
- [ ] Published with version and documentation.
- [ ] Scheduled maintenance review.
