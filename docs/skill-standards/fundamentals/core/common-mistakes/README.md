# Common mistakes

**Layer:** universal fundamentals. **Mode:** reference.

These are the failure modes that make skills unreliable, bloated, or ineffective. They are grouped by the kind of problem they create. Avoid them. When reviewing a skill, check it against this list.

---

## The mistake groups

- [`bloat.md`](./bloat.md) — sprawl, sediment, and duplication. The skill gets too big or repeats itself.
- [`weak-guidance.md`](./weak-guidance.md) — no-op instructions, vague completion criteria, over-reliance on priors, and guideline soup. The skill fails to change behavior.
- [`boundary-mistakes.md`](./boundary-mistakes.md) — upfront setup overload, hidden dependencies, over-configuring, under-declaring scope, and mixing invocation concerns. The skill's boundaries are unclear.
- [`tooling-and-shape-mistakes.md`](./tooling-and-shape-mistakes.md) — adapter tunnel vision and wrong type for the job. The skill uses the wrong tool or the wrong shape.
- [`workflow-mistakes.md`](./workflow-mistakes.md) — manual in disguise and premature completion. The skill is followed mechanically or rushed.
