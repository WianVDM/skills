# Core fundamentals

**Layer:** universal fundamentals. **Mode:** reference.

Universal rules every skill should satisfy. These are about what a skill is, when to create one, and how to keep it focused and predictable.

---

## Topics

- [`what-is-a-skill/`](./what-is-a-skill/) — the skill contract, delegation boundary, predictability, and pointers to deeper topics.
- [`when-to-create-a-skill/`](./when-to-create-a-skill/) — skill vs. script, MCP server, prompt template, or documentation change.
- [`structure/`](./structure/) — `SKILL.md` structure, frontmatter, progressive disclosure, and optional directories.
- [`context-budget.md`](./context-budget.md) — context load, description length, and when a skill pays its way.
- [`lifecycle/`](./lifecycle/) — decide → design → draft → validate → test → iterate → publish → maintain → retire.
- [`form-and-style/`](./form-and-style/) — instruction-heavy, guideline-heavy, and hybrid forms; completion criteria; leading words; pruning.
- [`common-mistakes/`](./common-mistakes/) — sprawl, sediment, duplication, weak guidance, boundary mistakes, tooling mistakes, and workflow mistakes.
- [`failure-recovery/`](./failure-recovery/) — detecting silent and loud failures, iterating without over-correcting, splitting, pruning, and retiring.
- [`examples/`](./examples/) — example skills to study.
