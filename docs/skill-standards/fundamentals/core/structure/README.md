# Structure

**Layer:** universal fundamentals. **Mode:** reference.

These documents describe the physical and organizational structure of a skill: the `SKILL.md` file, its frontmatter, optional sibling directories, progressive disclosure, and harness-agnostic language.

## Topics

- [`SKILL.md`](./skill-md.md) — the required file that contains the skill's identity and body.
- [`frontmatter.md`](./frontmatter.md) — the YAML frontmatter that declares identity, routing, and invocation mode.
- [`optional-directories.md`](./optional-directories.md) — the `references/`, `subagents/`, `scripts/`, and `assets/` directories.
- [`progressive-disclosure.md`](./progressive-disclosure.md) — how to move detail behind context pointers so the top level stays legible.
- [`harness-agnostic-language.md`](./harness-agnostic-language.md) — how to write skills that do not assume a specific harness or project.
- [`readme-file.md`](./readme-file.md) — the purpose of the `README.md` file inside a skill directory.
